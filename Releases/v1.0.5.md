# v1.0.5

## Highlights
- **Flexible Weight Randomization** — per-section and per-question randomizers for granular control
- **Context-Aware Smart Detection** — all smart detections now distinguish between similar terms to avoid false positives

## Flexible Weight Randomization

Three levels of randomization for maximum flexibility:

### Randomize All (Header Button)
- One-click randomizes weights for ALL questions on the form
- Located in the "Configure Weights (%)" panel header

### Per-Section Randomizer
- Questions grouped by form page/section
- Each section has its own randomize button (🔀)
- Only randomizes weights within that specific section
- Useful when you want different MCQ sections to have independent weight distributions

### Per-Question Randomizer
- Each question card has its own randomize button
- Located in the header of each question's weight configuration
- Randomizes only that specific question's weights
- Useful when you only want to randomize one question without affecting others

## Context-Aware Smart Detection

All smart detection patterns have been upgraded to use context-aware matching. The detection engine now intelligently distinguishes between similar terms to avoid false positives.

### Occupation Detection (Enhanced)
- **Detected**: "What is your occupation?", "Current profession", "Employment status"
- **Filtered**: "Comments about work flexibility", "Work-life balance feedback", "Feedback on work"

Detection logic now checks:
1. False positive patterns (comments, feedback, reflections about work)
2. Strong indicators (occupation, profession, job title standalone)
3. Medium indicators with context (employer, company, industry, "where do you work")

### Name Detection (Enhanced)
- **Detected**: "Your name", "Full name", "Participant name", "Name of student"
- **Filtered**: "Company name", "Product name", "File name", "Event name", "Account name", "Network name", "Login name"

Detection logic now checks:
1. "Name of" patterns with organizational exclusions (company, school, department, event, file, etc.)
2. Possessive indicators (your, my, the participant's, the respondent's)
3. Direct name field patterns (first name, last name, middle name, full name)

### Address Detection (Enhanced)
- **Detected**: "Your address", "Home address", "Current address", "what city", "your zip code"
- **Filtered**: "Country of birth" (→ nationality), "Favorite city", "City of birth", "ZIP code of an area", "birthplace country"

Detection logic now checks:
1. Country context - distinguishes address from nationality/birth origin questions
2. City context - excludes favorite/dream/city of birth questions
3. ZIP context - excludes area-related questions
4. Possessive indicators (your, what, which) before labeling as address

```javascript

  // 1. False positives - reject patterns that mention the term but aren't asking for it
  // 2. Strong indicators - standalone keywords that clearly indicate the field
  // 3. Medium indicators - keywords with context (possessives, question markers)
  return boolean;
}
```
