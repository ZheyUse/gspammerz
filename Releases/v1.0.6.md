# v1.0.6

## Highlights
- **Grid Row Weighting** - Google Forms grids now separate row prompts from answer columns, with independent sliders per row
- **Local Config Saving** - Auto Name, Auto Address, and Auto Nationality settings now persist locally across refreshes/restarts
- **Address Detection Fixes** - concise address labels like "Town", "Region", "Country", and "Postal / Zip Code" are recognized in address field groups
- **Submission Diagnostics** - clearer debugging for 400 causes, email collection checks, sentinel rows, and no-cors status limits
- **Native Git Updater** - update modal can now call a registered local updater host to run Git from the real repository root

## Grid Detection and Weighting

Grid handling was rebuilt so rows and columns are treated correctly:

- Grid rows are stored as row prompts, not answer choices
- Grid columns are used as the answer/weight slider values
- Each grid row gets its own row card
- Each grid row has its own randomizer button
- Each grid row has its own independent weight sliders
- Submissions pick an answer per row using that row's configured weights

Example:

```text
Question: Food Rating
GRID

Row 1: Pizza
  5 Great
  4 Good
  3 Neutral
  2 Bad
  1 Very Bad

Row 2: Chicken
  5 Great
  4 Good
  3 Neutral
  2 Bad
  1 Very Bad
```

### Grid Parser Improvements

- Fixed Google Forms internal type mapping for grid-like structures
- Added `gridRows` and `gridRowIds` to parsed grid questions
- Improved row-title extraction from nested Google Forms data
- Added DOM fallback for visible grid row labels when raw data is incomplete
- Fixed row labels falling back to `Row 1`, `Row 2`, etc. when real labels exist

## Local Config Saving

Auto configuration settings are now saved locally:

- Auto Name settings
- Auto Address settings
- Auto Nationality settings
- Include Extension toggle for Auto Name
- Address country/region/locality choices
- Nationality pool and match-local settings

Storage behavior:

- Uses `chrome.storage.local` when available
- Falls back to `localStorage` if needed
- Restores saved configs before rendering the UI
- Saves config changes through `updateState()`

### Manifest Update

Added the `storage` permission:

```json
"permissions": [
  "activeTab",
  "storage"
]
```

After updating, reload the unpacked extension in `chrome://extensions`.

## Address Smart Detection

Address detection now understands compact address test forms better.

Newly recognized examples:

- `Location`
- `Home Location`
- `Place of Residence`
- `Street Number and Street Name`
- `City / Municipality`
- `Town`
- `Region`
- `Postal / Zip Code`
- `Country`

The detector is now cluster-aware: when multiple address-like fields appear together, short labels like `Town`, `Region`, and `Country` are treated as address components without requiring words like "your" or "what".

## Submission Debugging

Submission diagnostics were improved:

- Fixed `emailCheck is not defined` crash
- Added safer email collection pre-flight checks
- Added diagnostics for likely 400 causes
- Added sentinel-row diagnostics for required grid rows
- Fixed `[object Object]` console output by stringifying debug details
- Reduced noisy no-cors warning stack traces
- Hardened payload logging for non-string FormData values

## Google Forms Payload Fixes

- Required grid row entries now receive matching answer values
- `_sentinel` fields are no longer treated as answer inputs
- Grid row values are applied to the correct row entry IDs
- Split date/time payload normalization is preserved
- Required empty entry fallback logic is safer

## UI Fixes

- Grid questions show row-card configuration instead of one shared slider group
- Row labels are visible beside each row card
- Grid row randomizers update only that row
- Auto Name Extension warning now reads from the saved Auto Name config
- Include Extension no longer appears disabled after saving and refreshing

## Native Update Helper

The update checker now supports two update paths:

- New manifest version: shows a version update
- Same version with new commits: shows a commit update list

For cloned repositories, `Update Now` can call a Chrome Native Messaging host:

- Native host name: `com.zheys.spammerz.updater`
- Host files live in `spammerz/native`
- The helper runs Git from the parent repo root, not just the loaded `spammerz` extension folder
- The helper uses `git fetch --prune` and `git pull --ff-only`
- Dirty working trees are blocked so local edits are not overwritten

Setup files:

- `spammerz/native/host.js`
- `spammerz/native/spammerz-native-host.cmd`
- `spammerz/native/install-host.cmd`
- `spammerz/native/install-host.ps1`
- `spammerz/native/uninstall-host.cmd`
- `spammerz/native/uninstall-host.ps1`

## Upgrade Notes

1. Reload the unpacked extension in `chrome://extensions`.
2. Refresh any open Google Form tabs.
3. Re-open Auto Name, Auto Address, or Auto Nationality once to confirm saved settings.
4. For grid forms, verify each row now has its own sliders and randomizer.
5. For automatic Git updates, run `spammerz/native/install-host.cmd` once and paste the extension ID from `chrome://extensions`.
